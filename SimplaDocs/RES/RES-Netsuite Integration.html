🔧 Technical Implementation Specifications

Authentication

Custom OAuth 1.0 workaround required - standard Coda Packs don't support OAuth 1.0. David requested special approval from Coda for custom pack.

Data Synchronization

Sync by task names (not IDs) due to NetSuite structure. More fragile than ID-based matching. Coda document IDs stored in NetSuite for reverse lookup.

User Identification

Email matching using @res.us addresses as common identifier. All personnel must be in Coda workspace for people chip functionality.

Performance Constraints

60-second timeout limit for pack actions. Complex operations may need external hosting. Document-level snapshots can cause delays.

Current Limitations

GET-only endpoints (no POST). Apollo approval system incomplete. Manual triggering required (no automatic hourly sync yet).

Error Handling

Dual error system: Admin errors (technical) and User errors (friendly). Status tracking via JSON objects in output columns.

User Workflow Requirements

1. Make Changes: Update task status in Coda (e.g., "Not Started" → "In Progress")

2. Wait: 30 seconds to 2 minutes for Coda snapshot creation

3. Sync: Click "Sync Tasks" button to send changes to Apollo

4. Track: Monitor status in Change Request columns

5. Pull Updates: Periodically check for approval/rejection status
