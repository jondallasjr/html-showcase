Core Workflows

1. User Login & Data Loading

1. User selects role (PM/RM/Employee/Admin) 2. User authenticates with credentials 3. System identifies user's region/department 4. UI loads user-specific projects from Supabase: - PM: Loads projects where user is PM - RM: Loads resource requests from user's department - Employee: Loads assignments where user is assigned - Admin: Access to all data 5. UI displays role-appropriate dashboard

2. Project Creation (PM)

1. PM clicks "New Project" ‚Üí Opens project creation modal 2. PM enters project details (Name, Delivery Method, State, etc.) 3. PM selects WBS template (based on Delivery Method) 4. System creates project in Supabase 5. System optionally syncs to NetSuite (creates NetSuite project) 6. System populates WBS from template 7. PM can now add cost items and milestones

3. Resource Request & Assignment (PM ‚Üí RM)

1. PM creates cost item of type "Labor" or "Equipment" ‚Üí Creates resource request 2. Resource request saved to Supabase with status "Unfilled" 3. RM views unfilled requests filtered by their department 4. RM selects request ‚Üí Opens assignment modal 5. RM selects actual resource (employee/equipment) 6. RM creates weekly assignments (with hours per week) 7. System validates capacity (checks for over-allocation) 8. Assignment saved to Supabase 9. Resource request status updated to "Filled" 10. PM notified of assignment

4. Progress Reporting (Employee ‚Üí PM)

1. Employee views weekly schedule 2. Employee clicks "Add Progress Report" for an assignment 3. Employee submits report with: - Status update (On Schedule, Behind, Ahead, Blocked) - Progress notes - Optional hour adjustment request 4. Report saved to Supabase with status "Pending" 5. PM notified of new report 6. PM reviews report in Progress Reports tab 7. PM approves/denies hour adjustment 8. If approved: System adjusts future assignments 9. PM provides feedback ‚Üí Employee notified

5. NetSuite Sync (Work Items)

1. PM creates/edits work items in Custom UI 2. Changes saved to Supabase 3. PM clicks "Sync to NetSuite" button 4. System sends work items to NetSuite via Apollo API 5. NetSuite returns task IDs and revision IDs 6. System stores NetSuite IDs in Supabase 7. For subsequent updates: System submits Task Change Request to Apollo 8. PM tracks TCR status in UI 9. If approved: NetSuite updates ‚Üí System pulls latest data
