üîß Technical Documentation

Architecture: Custom JavaScript UI ‚Üí Supabase Backend ‚Üí NetSuite/Apollo Integration

Data Storage Strategy

Supabase (Official Record)

  • Projects
  • Work Items (even unsent to NetSuite)
  • Milestones (separate table, displayed with Work Items)
  • Cost Items
  • Assignments
  • Progress Reports
  • Contract History
  • Generic Resource Rates

NetSuite (Official Record)

  • Projects (master data)
  • Work Items (basic: Name, Dates)
  • Employees
  • Vendors
  • Project Actuals
  • Regions, States, Delivery Methods
  • WBS Templates

Key Workflows

1. Project Creation

PM clicks "+ New Project" ‚Üí Fills details ‚Üí Saves to Supabase ‚Üí Optionally syncs to NetSuite ‚Üí Auto-navigates to Project Details

2. Resource Request & Assignment

PM adds Labor/Equipment cost item ‚Üí Creates resource request ‚Üí RM views unfilled requests ‚Üí Assigns employee ‚Üí System validates capacity ‚Üí Warns if over-allocated ‚Üí Assignment saved ‚Üí Request marked "Filled"

3. Progress Reporting

Employee submits progress report ‚Üí PM/RM reviews ‚Üí Approves/denies adjustments ‚Üí If approved: Future assignments auto-adjusted ‚Üí Employee notified

4. NetSuite Sync

PM creates/edits work items ‚Üí Saves to Supabase ‚Üí Clicks "Sync to NetSuite" ‚Üí Sends to Apollo API ‚Üí NetSuite returns IDs ‚Üí For updates: Creates Task Change Request ‚Üí Tracks TCR status ‚Üí When approved: Pulls latest data
