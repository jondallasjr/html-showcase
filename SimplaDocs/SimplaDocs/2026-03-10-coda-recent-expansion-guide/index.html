The New Core

Superhuman Go & the Agent Architecture

Go is the most strategically important product in the suite. It doesn't wait for prompts — it observes what you're working on and acts.

  • Proactive, context-aware AI that works everywhere Grammarly's browser extension reaches — across 1M+ apps and websites
  • Multi-agent architecture with connector agents (Google Workspace, Outlook, Jira, Confluence), Grammarly writing agents, and partner agents
  • Agent Store launched alongside Go — a marketplace for installable agents (Box, Gamma, Fireflies, Radical Candor, Wayground, and growing)
  • Built on Coda Packs SDK — agents are essentially Packs extended with skills, screen annotations, fetcher tools, MCP, and indexing
Hard Truth
Coda Brain was wound down with the October 2025 Superhuman launch. Its learnings were folded into Go. Brain's replacement lives outside Coda — in a cross-app agent layer, not in your docs.
How Go's agent architecture works
  • Connector agents bridge Go to external services — Google Workspace, Outlook, Jira, Confluence. They sync data and enable cross-app workflows.
  • Grammarly writing agents provide contextual writing assistance using the 16-year-old writing engine.
  • Partner agents are built by third-party developers via the Agents SDK. Published through the Agent Store.
  • Screen annotations are Grammarly-style inline suggestions — agents can suggest text rewrites directly on any webpage.
  • Skills are custom behaviors defined with prompts and tool configurations via pack.addSkill().
  • Agents can call external LLMs (like Gemini) and connect to MCP servers with pack.addMCPServer().
Coda Brain vs. Superhuman Go
  • Brain (launched June 2024): RAG-based enterprise AI search, powered by Snowflake Cortex, permission-aware, lived inside Coda docs. Ingested data from 500+ Packs.
  • Go (launched Oct 2025): Cross-app AI assistant, works everywhere via browser extension, multi-agent architecture, partner ecosystem.
  • Brain was Coda-confined. Go is platform-agnostic. The trade-off: more reach, less doc-native integration.
  • Brain pilots were wound down. A v2.0 shipped March 2025 before being absorbed. The technology wasn't lost — but the product surface shifted dramatically.
Developer Reality

Coda AI & Developer Tools Today

The in-doc AI features remain active and continue evolving. But there are notable gaps — and they matter.

  • AI Chat — Side-panel conversational AI with doc context
  • AI Assistant — Content generation via / commands
  • AI Column — Apply prompts at scale across table rows
  • AI Block — Page-level summarization
  • AI Reviewer — Inline feedback as comments
  • AI Pattern Fill — Auto-complete patterns in tables
  • 600+ Pack integrations — Connected to external services for AI context
Notable Gap
Coda AI is not particularly good at helping with Coda formulas — a Coda team member has acknowledged this. ChatGPT and Claude currently provide better formula assistance than Coda's own AI.
MCP (Model Context Protocol) support
  • Community MCP servers — Multiple unofficial servers (TypeScript and Python) bridge AI assistants like Claude Desktop or Cursor to the Coda API for natural-language read/write operations.
  • Official SDK MCP support — The Agents SDK supports MCP natively with pack.addMCPServer(). Currently limited to one MCP server per agent, Streamable HTTP transport only.
  • This means Coda data can be a context source for external AI tools — not just Coda's own AI features.
Recent Coda-specific updates
  • Two-way sync Packs — push changes back to Salesforce, Jira
  • Table locking with fine-grained permissions (April 2025)
  • Grids — simple layout tables
  • Timeline improvements
  • Admin AI analytics
  • Sync pages
  • Enhanced importers from Confluence, Notion, Airtable
  • Custom working days for formulas
The Evolution

The Agents SDK (Closed Beta)

Packs are evolving into agents. Same SDK, new capabilities. But there's a catch: most developers can't build them yet.

Packs → Agents: What's New

  • Skills — Custom behaviors defined with prompts and tool configurations via pack.addSkill()
  • Screen annotations — Grammarly-style text rewrite suggestions on any webpage
  • Fetcher tools — HTTP requests to external APIs, accessible to the agent runtime
  • MCP server tools — Connect to external MCP servers with minimal code
  • Indexing — Semantic search over sync table data for contextual agent responses
  • External LLM calls — Agents can invoke models like Gemini directly
Access Gate
The Agents SDK is in closed beta. Only select partners can build, test, and publish agents. The timeline for broader access is unclear. This is the single biggest bottleneck for Coda developers right now.
What stays the same from Packs
  • Same @codahq/packs-sdk framework
  • Same authentication patterns
  • Same sync table architecture
  • Pack skills translate directly to agent skills
  • Existing Pack development experience is the best preparation for agent development
Current known limitations
  • Closed beta — no self-serve access for most developers
  • One MCP server per agent (current limitation)
  • Streamable HTTP transport only for MCP
  • Agents are not yet available natively inside Coda docs — they live in Superhuman Go
  • Agent Store review process is not yet publicly documented
  • No public timeline for general availability

New Framing

The Competitive Landscape Has Fundamentally Changed

It's no longer "Coda vs. Notion." It's Superhuman vs. Microsoft Copilot vs. Google Gemini as AI orchestration layers for work.

Existential Risk
Superhuman relies on Google and Microsoft APIs for its overlay approach. Platform owners could restrict access or replicate the entire orchestration layer with Gemini and Copilot. This is the core bet.
The unique differentiator

Grammarly's 16-year installed base across 1M+ apps is what no competitor can replicate quickly. The browser extension is embedded in workflows across every major web application. This gives Superhuman Go a deployment vehicle that Microsoft (desktop-first) and Google (own-apps-first) don't have.

The bet: being platform-agnostic and already embedded everywhere is more valuable than owning the productivity suite yourself.
